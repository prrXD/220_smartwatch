# 小组验收报告

## SmartWatch产品概述

SmartWatch 是一款基于 **STM32F103C8T6** 微控制器的智能手表，搭载 **FreeRTOS** 实时操作系统。它集成了 OLED 显示屏、六轴惯性传感器、蓝牙通信模块，支持时间显示、运动监测、传感器数据查看、蓝牙数据同步等功能。

### 系统架构

```
 ┌─────────────────────────────────────────────────┐
 │                   应用层                         │
 │  ┌─────────┐ ┌──────────┐ ┌───────┐ ┌────────┐ │
 │  │ 表盘界面 │ │ 传感器监测 │ │ 运动  │ │ 设置  │ │
 │  │          │ │ (MPU6050) │ │ 计步器 │ │ 管理  │ │
 │  └────┬─────┘ └────┬─────┘ └───┬───┘ └───┬────┘ │
 ├───────┴────────────┴───────────┴─────────┴──────┤
 │              FreeRTOS 内核 (CMSIS RTOS v1)       │
 │  ┌──────────┐ ┌────────┐ ┌────────┐ ┌────────┐ │
 │  │ 时间任务 │ │ 传感器 │ │ 显示   │ │ 蓝牙   │ │
 │  │ (1s)    │ │(200ms) │ │(20ms)  │ │(50ms)  │ │
 │  └──────────┘ └────────┘ └────────┘ └────────┘ │
 ├─────────────────────────────────────────────────┤
 │              STM32 HAL 硬件抽象层                 │
 ├──────────┬────────┬─────────┬────────┬──────────┤
 │  SSD1306 │MPU6050 │  HC-05  │旋转编码器│ STM32F1 │

 │ (I2C1)   │(I2C1)  │(USART2) │(PA6/7) │  外设   │
 └──────────┴────────┴─────────┴────────┴──────────┘
```

## 硬件规格

| 组件 | 型号/规格 | 接口 |
|------|-----------|------|
| **主控 MCU** | STM32F103C8T6 (ARM Cortex-M3, 72MHz) | — |
| **显示屏** | SSD1306 OLED 128×64 像素, 单色 | I2C1 (PB6/PB7) |
| **六轴传感器** | MPU6050 (加速度计 ±2g + 陀螺仪 ±250°/s) | I2C1 (与 OLED 共用) |
| **蓝牙模块** | HC-05 (蓝牙 2.0, UART 9600 波特率) | USART2 (PA2/PA3) + DMA |
| **旋转编码器** | 机械增量式编码器 (x4 计数模式) | TIM3 (PA6/PA7) |
| **按键 1** | 菜单/确认/返回 | PB10 (内部上拉,低电平有效) |
| **按键 2** | 计步器暂停/恢复 | PA0 (内部上拉,低电平有效) |
| **HC-05 状态** | 蓝牙连接状态指示 | PB0 (高电平=已连接) |
| **晶振** | 8MHz HSE → PLL x9 = 72MHz | PD0/PD1 |
| **Flash 存储** | 64KB (设置保存于最后一页: 0x0800FC00) | — |
| **RAM** | 20KB | — |

| 引脚 | 功能 | 用途 |
|------|------|------|
| PA0 | GPIO_Input | 按键 2 (计步器控制) |
| PA2 | USART2_TX | HC-05 蓝牙发送 (DMA) |
| PA3 | USART2_RX | HC-05 蓝牙接收 (DMA) |
| PA6 | TIM3_CH1 | 旋转编码器 A 相 |
| PA7 | TIM3_CH2 | 旋转编码器 B 相 |

| PB7 | I2C1_SDA | OLED + MPU6050 数据 |
| PB10 | GPIO_Input | 按键 1 (菜单/确认) |
| PA13 | SWD_SWDIO | 调试接口 |
| PA14 | SWD_SWCLK | 调试接口 |

## 快速开始

### 开机

连接电源或 USB 供电后，手表自动启动。屏幕点亮后依次执行：

1. 系统时钟初始化 (72MHz)
2. OLED 显示屏初始化
3. MPU6050 传感器检测
4. HC-05 蓝牙模块初始化
5. FreeRTOS 任务启动

启动完成后进入 **表盘界面**（主界面）。

### 界面导航

手表采用三级菜单导航架构：

- **L0 — 主页面**: 5 个页面循环切换（表盘、传感器、运动、设置、系统信息）
- **L1 — 子菜单**: 进入某个功能页面的详细选项列表
- **L2 — 详情/编辑**: 查看详情或调整参数

## 操作指南

### 交互方式

| 操作 | 按键/组件 | 作用 |
|------|-----------|------|
| **顺时针旋转** | 旋转编码器 | 向下/向右移动光标；增加数值 |
| **逆时针旋转** | 旋转编码器 | 向上/向左移动光标；减少数值 |
| **短按 (<500ms)** | 按键 1 (PB10) | 确认选择/进入下一级 |
| **长按 (≥500ms)** | 按键 1 (PB10) | 返回上一级/快速回到表盘 |
| **短按** | 按键 2 (PA0) | 在计步器 App 中暂停/恢复计步 |

### 主页面切换

在表盘界面旋转编码器，可在以下 5 个主页之间切换：

1. **表盘** — 时间、日期、步数
2. **传感器** — MPU6050 姿态角、加速度、温度
3. **运动** — 步数、运动距离、卡路里消耗
4. **设置** — 时间设置、蓝牙配对、亮度、恢复出厂
5. **系统信息** — 电量、固件版本、蓝牙 MAC

## 蓝牙通信

### HC-05 模块

- **接口**: USART2 (PA2 TX, PA3 RX) + DMA
- **波特率**: 9600
- **状态检测**: PB0 (高电平 = 已连接)

### 传感器数据上报

蓝牙每 **3 秒**自动发送一次传感器数据，格式为文本行：

```
HH:MM:SS  ax:+X.XX ay:+Y.YY az:+Z.ZZ  gx:+X.X gy:+Y.Y gz:+Z.Z  pitch:+P.P roll:+R.R  temp:T  steps:12345
```

### 时间同步

手机 app 可通过蓝牙发送时间同步命令，两种格式：

1. **文本格式**: 发送 `T` 开头 + 时间字符串，例如 `T14:30:00`
2. **二进制帧**: AA 02 07 HH MM SS YY CHK 55 (AA 帧头, 02=命令ID, 07=数据长度, CHK=校验, 55=帧尾)

手表收到时间同步后自动更新系统时间。

### 异常恢复

如果 HC-05 出现异常（指示灯常亮、无法连接）：
- 系统自动检测 STATE 引脚状态
- 连续 60 秒无变化且未收到时间同步 → 触发硬件复位
- 复位方式：将 TX (PA2) 拉低 30ms 产生 BREAK 条件

## 电源管理

- **供电方式**: USB 或电池供电
- **Flash 设置保存**: 时间、亮度等设置自动保存到 STM32F103 内置 Flash 的最后页面 (地址 0x0800FC00)
- **数据格式**: 包含魔数 (0xA5A5A5A5) 和校验和的完整结构，防止写入损坏

## 故障排除

| 问题 | 可能原因 | 解决方法 |
|------|---------|---------|
| 屏幕无显示 | OLED 初始化失败 | 检查 I2C1 (PB6/PB7) 连接 |
| 传感器无数据 | MPU6050 未检测到 | 检查 I2C 连接和 AD0 引脚 (应为 GND) |
| 蓝牙无法连接 | HC-05 异常 | 系统自动复位；可尝试重新上电 |
| 步数不准确 | 阈值不合适 | 可调整 `PEDO_VAR_THRESHOLD` 编译参数 |
| 时间不保存 | Flash 写入失败 | 检查 Flash 最后一页是否被代码占用 |

## 创新点

### 一、系统架构精细化：RTOS调度从“可用”到“科学分级”
原要求仅需实现FreeRTOS多任务划分与调度，本设计在此之上做了量化优化：
1. **任务周期分级调度**：为不同任务明确了精准的运行周期（显示任务20ms、传感器采集200ms、蓝牙通信50ms、时间更新1s），按实时性需求分配CPU资源，既保证了UI流畅度，又兼顾了传感器与通信的响应速度，调度策略更科学。
2. **标准分层软件架构**：采用「应用层→FreeRTOS内核→HAL硬件抽象层→硬件外设」的四层架构，通过CMSIS RTOS v1封装FreeRTOS内核，代码耦合度低、可移植性强，超出了基础要求中“实现多任务管理”的范畴。
3. **硬件DMA性能优化**：蓝牙串口采用DMA收发机制，无需CPU频繁中断干预，降低主内核占用率，提升了通信稳定性与系统整体效率。

### 二、交互体验升级：构建完整的消费级手表交互体系
原要求仅需“时间+传感器两个显示页面”，扩展要求实现“旋转编码器+多级菜单”，本设计在此基础上搭建了完整的智能手表交互框架：
1. **三级深度导航架构**：设计了L0（5个主页循环切换）、L1（子菜单选项列表）、L2（详情查看/参数编辑）的三级导航逻辑，页面数量与层级深度远超要求，功能分区清晰、操作逻辑严谨。
2. **硬件级编码器驱动**：采用TIM3定时器的硬件编码器模式（x4计数），而非GPIO软件模拟脉冲，旋转识别精度更高、响应无延迟，交互手感与可靠性显著提升。
3. **精细化按键交互逻辑**：区分短按（<500ms，确认/进入）与长按（≥500ms，返回/快速回表盘）两种操作；双按键独立分工（菜单导航键+计步控制键），操作逻辑贴近商用智能手表的使用习惯。
4. **统一状态栏与视觉动效**：顶部8像素状态栏聚合电量、时间、蓝牙连接、传感器在线四类核心状态；图标菜单支持水平滑动动画，视觉体验的完成度远高于基础显示要求。

### 三、功能深度拓展：从基础实现到高可靠、高完整度方案
#### 1. 传感与计步：从“读数据”到“算法解算”
- 传感器不只是读取原始加速度/陀螺仪数值，还通过姿态解算得到俯仰角、横滚角，同时支持芯片温度读取；增加了传感器异常自动重试机制（3秒周期重检测），硬件容错性更强。
- 计步采用**加速度方差分析算法**，相比简单的阈值判断法抗干扰能力更强、计步准确率更高；在此基础上延伸出运动距离估算、卡路里消耗计算，支持暂停/恢复、步数清零，功能完整度远超基础计步要求。

#### 2. 蓝牙通信：从“单向同步”到“双向标准化协议”
原扩展要求仅需“蓝牙数据同步+上位机”，本设计定义了完整的双向通信体系：
- 上行：3秒周期结构化上报，单条数据包含时间、三轴加速度、三轴陀螺仪、姿态角、温度、步数全量信息，格式规范易解析。
- 下行：支持文本、二进制帧两种时间同步格式，二进制帧带帧头、命令ID、长度、校验、帧尾，通信抗干扰能力强。
- 新增**蓝牙异常自动复位机制**：60秒检测无响应则通过串口BREAK条件硬件复位模块，无需人工干预即可恢复，可靠性大幅提升。

#### 3. 新增掉电持久化能力
针对原设计“软件计时掉电丢失”的痛点，利用STM32内置Flash的最后一页存储时间、亮度等用户设置，加入魔数+校验和机制，避免异常写入导致的数据损坏，实用性显著增强。

### 四、硬件设计优化：资源复用与状态感知增强
1. **I2C总线高效复用**：OLED与MPU6050共用I2C1总线，仅占用2个IO引脚即可驱动两个外设，节省硬件资源、布线更简洁，充分发挥了I2C多设备通信的优势。
2. **全维度运行状态监测**：预留蓝牙连接状态引脚、传感器在线状态检测、电池电量可视化显示，系统运行状态可通过屏幕直观感知，不再是“黑盒运行”。
3. **标准化引脚规划**：SWD调试接口、外设引脚分配清晰，兼顾功能实现与开发调试需求，硬件预留扩展冗余。

### 五、工程化落地：从课程作业到可维护项目
1. **跨平台构建体系**：采用CMake+ARM GCC工具链构建项目，不绑定特定IDE，同时兼容STM32CubeMX项目生成，开发灵活性与可复现性更高。
2. **高度模块化代码架构**：驱动、UI、菜单、算法、蓝牙完全独立分文件实现，代码结构清晰，便于后续维护与功能扩展。
