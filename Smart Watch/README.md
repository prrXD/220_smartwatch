
# SmartWatch — 智能手表说明书

## 产品概述

SmartWatch 是一款基于 **STM32F103C8T6** 微控制器的智能手表，搭载 **FreeRTOS** 实时操作系统。它集成了 OLED 显示屏、六轴惯性传感器、蓝牙通信模块，支持时间显示、运动监测、传感器数据查看、蓝牙数据同步等功能。

### 系统架构

```
 ┌─────────────────────────────────────────────────┐
 │                   应用层                         │
 │  ┌─────────┐ ┌──────────┐ ┌───────┐ ┌────────┐ │
 │  │ 表盘界面 │ │ 传感器监测 │ │ 运动  │ │ 设置  │ │
 │  │          │ │ (MPU6050) │ │ 计步器 │ │ 管理  │ │
 │  └────┬─────┘ └────┬─────┘ └───┬───┘ └───┬────┘ │
 ├───────┴────────────┴───────────┴─────────┴──────┤
 │              FreeRTOS 内核 (CMSIS RTOS v1)       │
 │  ┌──────────┐ ┌────────┐ ┌────────┐ ┌────────┐ │
 │  │ 时间任务 │ │ 传感器 │ │ 显示   │ │ 蓝牙   │ │
 │  │ (1s)    │ │(200ms) │ │(20ms)  │ │(50ms)  │ │
 │  └──────────┘ └────────┘ └────────┘ └────────┘ │
 ├─────────────────────────────────────────────────┤
 │              STM32 HAL 硬件抽象层                 │
 ├──────────┬────────┬─────────┬────────┬──────────┤
 │  SSD1306 │MPU6050 │  HC-05  │旋转编码器│ STM32F1 │

 └──────────┴────────┴─────────┴────────┴──────────┘
```

## 硬件规格

| 组件 | 型号/规格 | 接口 |
|------|-----------|------|
| **主控 MCU** | STM32F103C8T6 (ARM Cortex-M3, 72MHz) | — |
| **显示屏** | SSD1306 OLED 128×64 像素, 单色 | I2C1 (PB6/PB7) |
| **六轴传感器** | MPU6050 (加速度计 ±2g + 陀螺仪 ±250°/s) | I2C1 (与 OLED 共用) |
| **蓝牙模块** | HC-05 (蓝牙 2.0, UART 9600 波特率) | USART2 (PA2/PA3) + DMA |
| **旋转编码器** | 机械增量式编码器 (x4 计数模式) | TIM3 (PA6/PA7) |
| **按键 1** | 菜单/确认/返回 | PB10 (内部上拉,低电平有效) |
| **按键 2** | 计步器暂停/恢复 | PA0 (内部上拉,低电平有效) |
| **HC-05 状态** | 蓝牙连接状态指示 | PB0 (高电平=已连接) |
| **晶振** | 8MHz HSE → PLL x9 = 72MHz | PD0/PD1 |
| **Flash 存储** | 64KB (设置保存于最后一页: 0x0800FC00) | — |
| **RAM** | 20KB | — |

### 引脚分配

| 引脚 | 功能 | 用途 |
|------|------|------|
| PA0 | GPIO_Input | 按键 2 (计步器控制) |
| PA2 | USART2_TX | HC-05 蓝牙发送 (DMA) |
| PA3 | USART2_RX | HC-05 蓝牙接收 (DMA) |
| PA6 | TIM3_CH1 | 旋转编码器 A 相 |
| PA7 | TIM3_CH2 | 旋转编码器 B 相 |
| PB0 | GPIO_Input | HC-05 连接状态 (STATE) |
| PB6 | I2C1_SCL | OLED + MPU6050 时钟 |
| PB7 | I2C1_SDA | OLED + MPU6050 数据 |
| PB10 | GPIO_Input | 按键 1 (菜单/确认) |
| PA13 | SWD_SWDIO | 调试接口 |
| PA14 | SWD_SWCLK | 调试接口 |

## 快速开始

### 开机

连接电源或 USB 供电后，手表自动启动。屏幕点亮后依次执行：

1. 系统时钟初始化 (72MHz)
2. OLED 显示屏初始化
3. MPU6050 传感器检测
4. HC-05 蓝牙模块初始化
5. FreeRTOS 任务启动

启动完成后进入 **表盘界面**（主界面）。

### 界面导航

手表采用三级菜单导航架构：

- **L0 — 主页面**: 5 个页面循环切换（表盘、传感器、运动、设置、系统信息）
- **L1 — 子菜单**: 进入某个功能页面的详细选项列表
- **L2 — 详情/编辑**: 查看详情或调整参数

## 操作指南

### 交互方式

| 操作 | 按键/组件 | 作用 |
|------|-----------|------|
| **顺时针旋转** | 旋转编码器 | 向下/向右移动光标；增加数值 |
| **逆时针旋转** | 旋转编码器 | 向上/向左移动光标；减少数值 |
| **短按 (<500ms)** | 按键 1 (PB10) | 确认选择/进入下一级 |
| **长按 (≥500ms)** | 按键 1 (PB10) | 返回上一级/快速回到表盘 |
| **短按** | 按键 2 (PA0) | 在计步器 App 中暂停/恢复计步 |

### 主页面切换

在表盘界面旋转编码器，可在以下 5 个主页之间切换：

1. **表盘** — 时间、日期、步数
2. **传感器** — MPU6050 姿态角、加速度、温度
3. **运动** — 步数、运动距离、卡路里消耗
4. **设置** — 时间设置、蓝牙配对、亮度、恢复出厂
5. **系统信息** — 电量、固件版本、蓝牙 MAC

## 功能说明

### 1. 表盘界面 (Watch Face)

默认主页面，显示：
- **状态栏**（顶部 8 像素）：电池百分比 🅱、当前时间 HH:MM、蓝牙连接状态 (BT)、IMU 传感器状态 (●)
- **日期行**：YYYY-MM-DD 星期
- **大号时间**：HH:MM:SS (16x24 像素字体)
- **底部按钮**：`菜单` | `设置`（通过光标选择，按编码器切换）

在表盘界面短按按键 1：
- 光标在「菜单」→ 进入 **图标菜单**
- 光标在「设置」→ 进入 **设置子菜单**

### 2. 图标菜单 (Icon Menu)

从表盘选择「菜单」后进入，显示 7 个应用图标（水平滑动动画效果）：

| 图标 | 功能 |
|------|------|
| 🔙 返回 | 退出图标菜单，返回表盘 |
| PEDO | 计步器 App（全屏显示步数、距离、卡路里） |
| MPU | 传感器 App（全屏显示 MPU6050 数据） |
| — | 预留 |
| — | 预留 |
| — | 预留 |
| — | 预留 |

操作：
- **旋转编码器**: 左右滑动切换图标
- **短按**: 选中当前图标对应的功能
- **长按**: 返回表盘

#### 计步器 App
显示当前步数、运动距离（米/公里）、消耗卡路里。
- **短按按键 1**: 清零步数
- **短按按键 2 (PA0)**: 暂停/恢复计步（左下角显示 `||` 表示暂停）
- **长按按键 1**: 返回图标菜单

#### 传感器 App
全屏显示 MPU6050 数据：姿态角（Pitch/Roll）、三轴加速度、温度。
- **长按按键 1**: 返回图标菜单

### 3. 传感器数据 (Sensors)

进入「传感器」主页或子菜单详情：
- **姿态** (Attitude)：Pitch（俯仰角）和 Roll（横滚角），单位 °
- **加速度** (Acceleration)：三轴 ax/ay/az，单位 m/s²
- **温度** (Temperature)：MPU6050 芯片内部温度，单位 °C

> 如果 MPU6050 未检测到，屏幕显示 "NOT FOUND"，系统每 3 秒自动重试。

### 4. 运动数据 (Sports)

- **今日步数**：从开机起累计的步数
- **距离**：根据步数估算的运动距离（每步约 0.7 米）
- **卡路里**：根据步数估算的消耗（每步约 0.04 kcal）

步数检测基于加速度方差分析算法，灵敏度可适应轻微手臂摆动。

### 5. 设置 (Settings)

进入「设置」页面后短按按键 1 进入子菜单，包含 4 项：

#### 时间设置 (Time Set)
按顺序设置：**年 → 月 → 日 → 时 → 分**
- 旋转编码器调整数值
- 短按确认当前字段并进入下一字段
- 设置完成后自动保存到 Flash

#### 蓝牙配对 (BT Pairing)
查看蓝牙连接状态（Connected/Disconnected）。

#### 亮度调节 (Brightness)
- 旋转编码器调节 OLED 亮度 (0–255)
- **实时生效**，调节时屏幕亮度立即变化
- 短按保存并返回

#### 恢复出厂设置 (Factory Reset)
- 选择 `CANCEL` 取消
- 选择 `CONFIRM` 确认重置所有设置为默认值

### 6. 系统信息 (System Info)

- **电池** (Battery)：当前电量百分比及图形化电池条
- **固件** (Firmware)：SmartWatch v1.0，MCU: STM32F103C8T6
- **蓝牙 MAC**：HC-05 模块信息，USART2 接口，9600 波特率

### 7. 视频播放 (Video)

系统内置了 5 帧 64×64 像素动画（约 10fps），可通过图标菜单或 App 页面触发播放（预留功能）。

## 状态栏说明

屏幕顶部 8 像素区域为状态栏，从左到右显示：

```
[🔋85%]    [10:30]        [BT] [●]
 ───────   ────────   ──────  ────
 电池电量   当前时间   蓝牙已   IMU 就绪
 百分比    HH:MM     连接    (实心=在线
                           空心=离线)
```

- **电池图标 + 百分比**: 0–100%
- **时间**: 24 小时制 HH:MM
- **BT**: 仅当蓝牙已连接时显示
- **● / ○**: 实心 = MPU6050 传感器正常，空心 = 未检测到

## 蓝牙通信

### HC-05 模块

- **接口**: USART2 (PA2 TX, PA3 RX) + DMA
- **波特率**: 9600
- **状态检测**: PB0 (高电平 = 已连接)

### 传感器数据上报

蓝牙每 **3 秒**自动发送一次传感器数据，格式为文本行：

```
HH:MM:SS  ax:+X.XX ay:+Y.YY az:+Z.ZZ  gx:+X.X gy:+Y.Y gz:+Z.Z  pitch:+P.P roll:+R.R  temp:T  steps:12345
```

### 时间同步

手机 app 可通过蓝牙发送时间同步命令，两种格式：

1. **文本格式**: 发送 `T` 开头 + 时间字符串，例如 `T14:30:00`
2. **二进制帧**: AA 02 07 HH MM SS YY CHK 55 (AA 帧头, 02=命令ID, 07=数据长度, CHK=校验, 55=帧尾)

手表收到时间同步后自动更新系统时间。

### 异常恢复

如果 HC-05 出现异常（指示灯常亮、无法连接）：
- 系统自动检测 STATE 引脚状态
- 连续 60 秒无变化且未收到时间同步 → 触发硬件复位
- 复位方式：将 TX (PA2) 拉低 30ms 产生 BREAK 条件

## 电源管理

- **供电方式**: USB 或电池供电
- **Flash 设置保存**: 时间、亮度等设置自动保存到 STM32F103 内置 Flash 的最后页面 (地址 0x0800FC00)
- **数据格式**: 包含魔数 (0xA5A5A5A5) 和校验和的完整结构，防止写入损坏

## 固件更新与开发

### 构建系统

项目使用 **CMake** 构建，配合 ARM GCC 工具链：

```bash
# 创建构建目录
mkdir build && cd build

# 配置 (Debug 模式)
cmake .. -DCMAKE_TOOLCHAIN_FILE=path/to/arm-none-eabi-gcc.cmake

# 编译
cmake --build .
```

### 主要源文件

| 文件 | 功能 |
|------|------|
| `Core/Src/main.c` | 主程序入口，外设初始化 |
| `Core/Src/freertos.c` | FreeRTOS 任务定义 (时间/传感器/显示/蓝牙) |
| `Core/Src/oled.c` | SSD1306 OLED 驱动 |
| `Core/Src/OLED_data.c` | 字体、图标、视频帧数据 |
| `Core/Src/smartwatch_ui.c` | UI 框架：5 个主页、状态栏、图标菜单、App 页面 |
| `Core/Src/menu.c` | 菜单状态机构：L0/L1/L2 三级导航 |
| `Core/Src/mpu6050.c` | MPU6050 传感器驱动 + 计步器算法 |
| `Core/Src/bluetooth.c` | HC-05 蓝牙通信、时间同步解析 |

### 编程/调试

- **调试接口**: SWD (PA13 SWDIO, PA14 SWCLK)
- **IDE 支持**: 可通过 STM32CubeMX 生成项目文件
- **串口调试**: 通过 USART2 (PA2/PA3) 输出传感器日志

### 依赖

- **STM32 HAL 库** (STM32F1xx HAL Driver)
- **CMSIS** (ARM Cortex 内核抽象层)
- **FreeRTOS** (通过 CMSIS RTOS v1 封装)
- **数学库** (m) — 用于 atan2f、sqrtf 等浮点运算

## 故障排除

| 问题 | 可能原因 | 解决方法 |
|------|---------|---------|
| 屏幕无显示 | OLED 初始化失败 | 检查 I2C1 (PB6/PB7) 连接 |
| 传感器无数据 | MPU6050 未检测到 | 检查 I2C 连接和 AD0 引脚 (应为 GND) |
| 蓝牙无法连接 | HC-05 异常 | 系统自动复位；可尝试重新上电 |
| 步数不准确 | 阈值不合适 | 可调整 `PEDO_VAR_THRESHOLD` 编译参数 |
| 时间不保存 | Flash 写入失败 | 检查 Flash 最后一页是否被代码占用 |
